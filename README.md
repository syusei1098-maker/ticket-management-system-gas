# ticket-management-system-gas
自社イベント用「リアルタイム・チケット管理システム」（GAS + Spreadsheet）
1. プロジェクト概要と開発背景
自社で定期開催している大規模顧客イベント「感謝祭」において、飲食ブースの注文・調理・受け渡し工程をデジタル化し、チームの連携を最大化させるために開発したフルスタックWebアプリケーションです。
解決した課題： 紙の伝票による「注文の聞き間違い」や「調理状況の不透明さ」の解消。
行列の発生を防ぐための「呼び出しのリアルタイム化」。
イベント終了後の「集計作業（在庫・売上）」の自動化。
コンセプト： 「顧客とスタッフに最高のチームワーク体験を提供する」
2. システム構成・使用技術
「現場で即座に修正・運用ができる柔軟性」と「導入コストの最小化」を両立するため、Google Cloudインフラを活用した構成を採用しました。
Frontend: HTML, CSS, JavaScript
Backend: Google Apps Script (GAS)
Database: Google Sheets（リアルタイムDBとして活用）
Security: アクセス権限管理（スタッフ用URLの秘匿化）、バリデーション処理
3. 業務フローと機能詳細（スクリーンショット参照）
現場の各役割に応じた4つの専用インターフェースを構築し、データの一貫性を保持しています。
ステップ
画面・機能
詳細説明
1
注文入力画面
顧客からの注文を直感的なUIで入力。即座にデータベースへ反映。
2
注文完了通知
注文番号・QRコードを自動発行し、顧客へ提示。
※件数制限はあるがメール配信も可能
3
キッチン管理画面
注文内容を時系列で表示。在庫の欠品状況をリアルタイムに更新・共有可能。
4
呼び出し状況画面
調理完了ボタンに連動し、顧客向けモニターの表示が自動更新。
5
受け渡し検証
QRコードまたは注文番号入力により、なりすましや渡し間違いを防止。
6
受け渡し完了
ステータスを「完了」に更新し、全画面から情報をクリア。
7
データ自動集計
全ての挙動がスプレッドシートへログ出力され、自動でグラフ化・分析。

4. 開発におけるこだわりと将来への展望
現場第一のUI設計： 繁忙期でも誤操作が起きないよう、大きなボタン配置と視認性の高い配色を採用しました。
セキュリティエンジニアとしての視点： 単なる「動くアプリ」ではなく、受け渡し時の「検証プロセス」を設けることで、データの整合性と実世界の安全性を両立させました。将来的には、この「現場の利便性を損なわない認証・認可」のロジックをクラウドセキュリティの領域（IAMやゼロトラスト）へ昇華させたいと考えています。

生産性の追求： GASの実行速度を補うため、フロントエンド側での非同期処理を工夫し、ストレスのない操作感を実現しました。
